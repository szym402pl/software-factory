# Phase 2 — Standard

For: new features, dependency decisions, common problems, "someone definitely built this before."

## Strategy

- 2-3 WebSearch queries
- Check npm, GitHub, SaaS alternatives
- Different vocabulary per query (builder's words, user's words, adjacent-community framing)

## Output

Short verdict with links:

```
Verdict: Use X | Adapt Y | Build minimum
- [X](link) — what it does, why it fits
- [Y](link) — alternative considered, why not
- What to build if nothing fits: <1-2 lines>
```
